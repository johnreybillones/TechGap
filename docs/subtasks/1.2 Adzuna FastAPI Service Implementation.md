For **Subtask 1.2: Adzuna FastAPI Service**, the goal is to build a **fast-burst data supplement** that seeds the `jobs_raw` table quickly during the first 1–2 days of the project, and acts as a **fallback** when the JobStreet scraper falls below the per-track minimum. Adzuna is **not the primary source** — JobStreet is. Adzuna's value is speed: it can deliver ~1,000–1,500 Philippine IT job listings via a clean API in minutes, giving the team enough data (~1,500 total threshold) to begin the NLP pipeline while the scraper continues running in the background.

## **🛠️ Subtask 1.2: Adzuna FastAPI Service Implementation**

### **1\. Developer Preparation**

Before coding, the developers need to secure access to the Adzuna API.

* **Action:** Sign up at the [Adzuna Developer Portal](https://developer.adzuna.com/).  
* **Required Keys:** `ADZUNA_APP_ID` and `ADZUNA_APP_KEY`.  
* **Target Region:** `ph` (Philippines).

### **2\. FastAPI Service Architecture and Role**

Adzuna should be run in **one of two scenarios:**

1. **Day 1–2 seed pull:** Run once at the start of the project to quickly populate the database with ~1,000–1,500 jobs while the Playwright scraper is being set up.
2. **Fallback trigger:** Automatically triggered if JobStreet's accumulated yield for a specific track falls below **300 jobs** by end of Week 1.

The service should be structured to handle "batches" of data. Since the Adzuna PH endpoint typically returns 50 results per page and has a total inventory of ~1,000–1,500 PH IT jobs, the script should collect **all available results** rather than targeting a fixed count.

#### **B. The Collector Logic (`collector.py`)**

note: *"Adzuna is a supplement, not the primary source. If Adzuna returns fewer than 300 jobs for a given track, that is acceptable — the JobStreet scraper will cover the remainder over time. The goal for Adzuna is to provide a fast initial seed so NLP pipeline development can begin in Week 1."*

This script will be triggered via a FastAPI endpoint. It handles the API requests and the initial save to Supabase.

import httpx  
import os  
from supabase import create\_client, Client

\# Initialize Supabase  
url: str \= os.environ.get("SUPABASE\_URL")  
key: str \= os.environ.get("SUPABASE\_KEY")  
supabase: Client \= create\_client(url, key)

async def fetch\_adzuna\_jobs(keyword: str, target\_count: int \= 300):  
    app\_id \= os.environ.get("ADZUNA\_APP\_ID")  
    app\_key \= os.environ.get("ADZUNA\_APP\_KEY")  
    base\_url \= f"https://api.adzuna.com/v1/api/jobs/ph/search"  
      
    collected\_count \= 0  
    page \= 1  
      
    while collected\_count \< target\_count:  
        params \= {  
            "app\_id": app\_id,  
            "app\_key": app\_key,  
            "results\_per\_page": 50,  
            "what": keyword,  
            "content-type": "application/json"  
        }  
          
        async with httpx.AsyncClient() as client:  
            response \= await client.get(f"{base\_url}/{page}", params=params)  
            data \= response.json()  
              
            jobs \= data.get("results", \[\])  
            if not jobs:  
                break \# Stop if no more jobs are found  
                  
for job in jobs:  
    job\_data \= {  
        "source": "adzuna",  
        "external\_id": job.get("id"),  
        "job\_title": job.get("title"),  
        "description": job.get("description"),       \# Core field for SBERT embeddings  
        "contract\_type": job.get("contract\_type"),  
        "seniority\_level": None,                   \# NULL until Week 2 NLP extraction (Task 2.4)  
        "company\_name": job.get("company", {}).get("display\_name"),  
        "location": job.get("location", {}).get("display\_name"),  
        "salary\_min": job.get("salary\_min"),         \# PRD-required contextual metadata  
        "salary\_max": job.get("salary\_max"),  
        "raw\_json": job   
    }  
                  
                \# Upsert into Supabase (prevents duplicates based on external\_id)  
                supabase.table("jobs\_raw").upsert(job\_data).execute()  
                  
            collected\_count \+= len(jobs)  
            page \+= 1  
            print(f"Collected {collected\_count} jobs for {keyword}...")

#### **C. The FastAPI Endpoint (`main.py`)**

This allows the team to trigger a "Fresh Pull" of data from a dashboard or a simple URL call.

Python  
from fastapi import FastAPI, BackgroundTasks

app \= FastAPI()

@app.post("/collect/adzuna")  
async def trigger\_collection(background\_tasks: BackgroundTasks):  
    \# List of keywords aligned with your 4 tracks  
    keywords \= \[
        \# CS-IS
        "Machine Learning", "AI Engineer", "Data Scientist",
        \# CS-GD
        "Unity Developer", "Game Programmer", "Unreal Engine",
        \# IT-WD
        "Full Stack Developer", "Frontend React", "Backend Developer",
        \# IT-NT
        "Network Administrator", "Cybersecurity", "Cloud Engineer"
    \]  
      
    for kw in keywords:  
        \# Run as background task so the API doesn't time out  
        background\_tasks.add\_task(fetch\_adzuna\_jobs, kw)  
          
    return {"status": "Collection started in background for all tracks."}

### **4\. Critical Developer Instructions**

1. **Keyword Strategy:** Developers must use keywords that specifically target your 4 curriculum tracks:  
   * **CS-IS:** "Machine Learning", "AI Engineer", "Data Scientist".  
   * **CS-GD:** "Unity Developer", "Game Programmer", "Unreal Engine".  
   * **IT-WD:** "Full Stack Developer", "Frontend React", "Backend Developer".  
   * **IT-NT:** "Network Administrator", "Cybersecurity", "Cloud Engineer".  
2. **Rate Limiting:** Adzuna has a limit on requests per minute. If the script fails with a `429` error, add `import time` and `time.sleep(2)` between page loops.  
3. **The "Raw JSON" Rule:** **Do not try to clean the data yet.** The developers must save the entire `job` object into the `raw_json` column. This is vital because the **Inference Engine** in Phase 2 will need to "re-read" the raw descriptions to find hidden skills.
